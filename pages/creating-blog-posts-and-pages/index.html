Prerequisites

  • Ruby 2.2.3
  • Git / Github Desktop
  • Knowledge of the Markdown language
  • A text editor (I recommend Atom)

  • A local clone of this repository, obtained with the following command:

    {% highlight bash %} git clone https://github.com/ImperialRobotics/imperialrobotics.github.io {% endhighlight %}

Creating a blog post

Crreaing a blog post is a simple matter of one (or four) commands once Ruby is installed, which is a nice feature courtesy Jekyll Bootstrap, the framework this site is based on. Below are the steps required to easily create a new blog post with little effort on your end.

Using Ruby

  1. Change your working directory to your local working copy of this repository:

    {% highlight bash %} cd imperialrobotics.github.io {% endhighlight %}

  2. Use "Rake" (Ruby Make) to create a new post with your desired title:

    {% highlight bash %} rake post title="your title goes here" {% endhighlight %}

  3. Open the newly created file, located at /_posts/<TODAY'S DATE>-<POST TITLE>.md. Its contents should look something like this:

    {% highlight yaml %} --- layout: post title: "post title" description: "" category: "" tags: [] --- { %include JB/setup% } {% endhighlight %}

  4. Edit the newly created file, filling out the description section with a short description of the new post between the quotes, the tags section by adding new tags, seperated by commas, in quotes, similar to this exapmle: tags: ["test", "post"], and the actual body content by placing your desired content TWO lines below the final line in the new file. This will require knowledge of the Markdown syntax as stated above in the Prerequisites section.

  5. Add your changes to staging:

    {% highlight bash %} git add * {% endhighlight %}

  6. ... and then commit them:

    {% highlight bash %} git commit -m "explanatory commit message goes here" {% endhighlight %}

  7. ... then push them globally when you're done, following the on screen prompts to complete the proces:

    {% highlight bash %} git push {% endhighlight %}
